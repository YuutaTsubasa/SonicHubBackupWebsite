接下來我要說明索尼克世代游戲目錄中的文件分別有哪些作用。
XXX.cfg -ConfigurationTool.exe的設置信息文件，里面的DefaultInput.cfg記錄著默認鍵位
保存鍵位我的win7在我的文檔的My Games\\Sonic Generations\\Saved Games里
cpkredir.ini -可用于修改SONICGMI軟件的MODS文件夾位置
movie文件夾 -游戲過場動畫裝在這里
Sound文件夾 -游戲的BGM背景音樂被封裝成CPK格式保存在這個文件夾中
disk文件夾 -索尼克世代最主要的游戲文件被封裝成CPK格式裝在這個文件夾中


在游戲解包時會遇到的格式
.CPK 最外層的游戲封包格式，可視作一個壓縮打包文件，
可用QuickBMS或CRI Packed File Maker解壓，CRI Packed File Maker還可用來封包


.ar.00,01,02,03,04.... CPK內第二層封包格式文件，
可使用artools或GenArcEd解包，GenArcEd封包


.arl 00,01,02等文件打包時會產生的文件，似乎有分卷與記號的作用，并不需要解包


.csb 音頻文件封裝包，文件中封裝了多個“.axx”的第二層格式音頻組合文件，
.axx還需要解壓（轉換）為adx格式（其中多個adx格式音頻文件可轉換成一個axx組合文件導入csb中）
.adx格式音頻并不方便播放，還需轉換成wav格式才好處理操作。


.gte 存在與00,01,02等文件中的模型配置文件，可通過gtexml.exe互轉為xml文本格式


.dds directX貼圖文件，可在Adobe PhotoShop中安裝DDS插件修改游戲貼圖，
在索尼克世代中支持DXT5 ARGB 8 BPP/interpolated alpha高質量貼圖算法格式與
1.5.5.5 ARGB 16bpp/unsigned壓縮圖片格式


.model 3D模型文件，一般用于索尼克人物模型，NPC模型，場景中敵人模型，道具模型等


.skl.hkx 3D模型骨骼文件


.hkx 3D模型可用的Havok動作文件


.material 3D模型貼圖的材質配置文件，比如反光有多強烈，本體顏色偏向哪，發不發光等……


.uv-anim 一般用于記錄索尼克，NPC，怪物等眼神的方向朝向哪邊


.xml 文本文件，一般用于記錄文件表列，配置信息


fco 特殊算法的文本文件，用于記錄NPC，OMOCHAO的字幕等
十六進制編輯字符對應是啥sonicretro有詳細碼表


.xncp 特殊算法的動畫文件，比如播放加載動畫黑白大小索奔跑動畫，
H.A攻擊出現的綠色鎖定圖標等等，至今無人破解，無法修改


.pfd 游戲場景地圖3D建模文件

然后是解包或制作MOD時會遇到的文件各種說明用處。




打開一個MOD的文件夾，里面長這樣，disk和sound里分別是CPK解壓后的文件夾，
旁邊的mod.ini文本文件記錄著mod信息
內容類似是如下的：

[Main]
IncludeDir0=\".\"（華麗無視）
IncludeDirCount=排序
UpdateServer=\"最新版本更新地址（更新用，可不寫）\"


[Desc]
Title=\"模組的名稱\"
Description=\"補丁的說明，介紹\"
Version=\"這是XXX版本\"
Date=\"創建日期\"
Author=\"補丁作者的名字\"
URL=\"關聯網址\"


[CPKs]（←一般沒有）


舉個例子


[Main]
IncludeDir0=\".\"
IncludeDirCount=1



[Desc]
Title=\"紅色版索尼克\"
Description=\"我制作的第一個MOD！體驗紅色版索尼克的爽快感！\"
Version=\"1.0\"
Date=\"2015.8.6\"
Author=\"度娘\"
URL=\"http://www.大屌.com\"

MOD.INI文件可以復制別人的再修改來參考一下，或者自己建個txt，
搞完后把文件尾綽名也連帶修改成MOD.INI


其中disk文件夾里面是這樣的cpk文件
mod文件夾的disk文件夾內的bb，bb2,3中只需要裝入修改過的文件即可




bb：主要裝地圖文件，場景配置文件
bb2：主要裝地圖文件，場景中敵人文件，即時過場文件，密室模型文件，界面文件
bb3：主要裝索尼克模型，配置，動作，語音文件


!archive-info.xml 出現在00,01,02文件夾中的文件管理整理文本，哪個文件分別用于哪里等


bb3文件說明：
#Sonic.ar.00 索尼克的配置文件，可修改其中的xml文件，讓索尼克空中飛行，
速度加兩倍快，鎖定攻擊超遠超快距離，跳躍跳得很高等。其中同樣的配置信息會在xml出現兩次，
是因為2D與3D時能力是不一樣的，分別會設置一遍2D和3D的配置。


#SonicClassic.ar.00 小索的配置文件


Sonic.ar.00，Sonic.ar.01 索尼克的游戲模型，動作，貼圖等都在這里面


SonicClassic.ar.00，SonicClassic.ar.01 小索的游戲模型，動作，貼圖等都在這里面


#Application.ar.00 管理劇情順序，新游戲開始進哪關，進哪個模式的HUB，
玩到哪個部分播放哪個過場動畫等


Languages文件夾 這個文件夾會出現在另外的bb與bb2中，里面裝著界面圖標圖片，
下面寫到bb與bb2時不再重復。


voices文件夾 里面裝著索尼克的語音csb文件


其他很多帶SONICXXXX.00,都是在不同場合會載入的動作文件。

然后是一些修改軟件的使用方法：


PS DDS插件 因為不同版本PS的DDS插件都不一樣，我就不一一提供了
另外也有很多查看圖軟件是支持查看DDS畫面和通道的，比如2345看圖王


有繪制貼圖經驗的可以直接在圖片上修改，半透明的圖片請記得添加圖片的透明通道。


CPK解壓工具QuickBMS


里面有2個exe，我們需要的是quickbms_4gb_files.exe
打開后是這也樣子的：


會直接從dos窗口中彈出選擇文件，首先得載入軟件目錄里的一個cpk.bms配置文件，雙擊即可，第二次會再次彈出選擇文件窗口，這時你就可以選擇需要解壓的CPK文件了




選擇好后還會再次彈出第三次選擇窗口，這次是要你選擇一個文件夾好放解壓的文件，
比如你選了解壓bb3.cpk后就新建一個bb3文件夾，把bb3.cpk的文件解壓到bb3文件夾中去
就在BB3文件夾里選擇保存即可，就會開始解壓。
bb2和bb因為文件巨大，會需要解壓很久，bb3文件小，速度快。
http://pan.baidu.com/s/1o6knCAe


一般解壓出來的文件就可以用來改再放進sonicGMI里了，
如果改好了的文件如果要封包回cpk的話（一般人很少這么做），就用Cri Packed File Maker






選擇一個文件夾，把一個文件夾里的所有文件打包成CPK格式文件




選Build Cpk file，選好要生成的cpk名稱，CPK如我圖中的設置，點Start to Build即可開始
生成新的CPK文件。搞成CPK理由應該只有防止SONICGMI出現無限加載狀況了，
防止出現無限加載的話只要打包成CPK就行了，當然你得備份。
http://pan.baidu.com/s/1mg1SDRE




artools-解壓00,01,02等
把00,01格式文件用鼠標按住，拖到ar0unpack.EXE上，
軟件會把解包文件夾生成在該文件目錄，比如你解壓的是1.ar.00，
解壓出來的就是1.ar.00.unpack文件夾，當然為了方便操作，
你可以把00,01,02等文件打開方式，直接選擇這個exe文件，
雙擊一下就能直接解壓，多方便。



artools：
http://pan.baidu.com/s/1jG1WIxw



打包00,01,02等文件軟件GenArcEd
打開Generations Archive Editor.exe后會有一個小窗口，把要打包的文件框起來選中拖進去后
選file-save as即可另存為新的00,01等文件，也可直接覆蓋mod的原版文件

Sonic Generations LVL master 0.9.0（libgens sonicglvl master）
索尼克世代的地圖編輯器，人物預覽查看器。
用于查看模型動作，編輯場景道具，把3DSMAX場景轉換成世代場景用
本體程序在SGLVL master 0.9.0\\bin\\SonicGLvl.exe




默認配置即可，熟練使用這個需要一定的3D基礎，
鏡頭旋轉與移動方法快捷鍵和3DSMAX操作方式相同，
載入地圖方法：從file里選擇open stage打開關卡舞臺，就拿之前說過的pla200植物星球來說好了
對應文件就是bb2里的#pla200.ar.00，打開后等待一定時間的關卡轉換即可載入







使用后舞臺的存緩是在SGLVL master 0.9.0\\cache文件夾里的，
請如果不需要再次載入這個地圖的話請手動刪掉存緩。
